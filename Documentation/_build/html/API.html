API Documentation¶

Model Creation¶

Model_Creation.Model_Creation(model)¶

This function creates the instance for the resolution of the optimization in Pyomo.

Parameters:model – Pyomo model as defined in the Micro-Grids library.

Initialize parameters¶

Initialize.Initialize_Demand(model, i, t)¶

This function returns the value of the energy demand from a system for each period of analysis from a excel file.

Parameters:model – Pyomo model as defined in the Model_Creation script.
Returns:The energy demand for the period t.
Initialize.Initialize_PV_Energy(model, i, t)¶

This function returns the value of the energy yield by one PV under the characteristics of the system analysis for each period of analysis from a excel file.

Parameters:model – Pyomo model as defined in the Model_Creation script.
Returns:The energy yield of one PV for the period t.
Initialize.Initialize_years(model, i)¶

This function returns the value of each year of the project.

Parameters:model – Pyomo model as defined in the Model_Creation script.
Returns:The year i.

Constraints¶

Constraints.Battery_Reposition_Cost(model)¶

This funtion calculate the reposition of the battery after a stated time of use.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Diesel_Comsuption(model, i, t)¶

This constraint transforms the energy produce by the diesel generator in to liters of diesel in each scenario i.This is done using the low heating value of the diesel and the efficiency of the diesel generator.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Diesel_Cost_Total(model, i)¶

This constraint calculate the total cost due to the used of diesel to generate electricity in the generator in each scenario i.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Energy_balance(model, i, t)¶

This constraint ensures the perfect match between the energy energy demand of the system and the differents sources to meet the energy demand each scenario i.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Financial_Cost(model)¶

This constraint calculate the yearly payment for the borrow money.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Initial_Inversion(model)¶

This constraint calculate the initial inversion for the system.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Max_Bat_in(model, i, t)¶

This constraint maintains the energy in to the battery, below the maximum power of charge of the battery for each scenario i.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Max_Bat_out(model, i, t)¶

This constraint maintains the energy from the battery, below the maximum power of discharge of the battery for each scenario i.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Max_Power_Battery_Charge(model)¶

This constraint calculates the Maximum power of charge of the battery. Taking in account the capacity of the battery and a time frame in which the battery has to be fully loaded for each scenario.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Max_Power_Battery_Discharge(model)¶

This constraint calculates the Maximum power of discharge of the battery. for each scenario i.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Maximun_Charge(model, i, t)¶

This constraint keeps the state of charge of the battery equal or under the size of the battery for each scenario i.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Maximun_Diesel_Energy(model, i, t)¶

This constraint ensures that the generator will not exceed his nominal capacity in each period in each scenario i.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Maximun_Lost_Load(model, i)¶

This constraint ensures that the ratio between the lost load and the energy Demand does not exceeds the value of the permisible lost load each scenario i.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Minimun_Charge(model, i, t)¶

This constraint maintains the level of charge of the battery above the deep of discharge in each scenario i.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Net_Present_Cost(model)¶

This function computes the sum of the multiplication of the net present cost NPC (USD) of each scenario and their probability of occurrence.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Operation_Maintenance_Cost(model)¶

This funtion calculate the operation and maintenance for the system.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Scenario_Lost_Load_Cost(model, i)¶

This constraint calculate the cost due to the lost load in each scenario i.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Scenario_Net_Present_Cost(model, i)¶

This function computes the Net Present Cost for the life time of the project, taking in account that the cost are fix for each year.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Solar_Energy(model, i, t)¶

This constraint calculates the energy produce by the solar panels taking in account the efficiency of the inverter for each scenario.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.State_of_Charge(model, i, t)¶

This constraint calculates the State of charge of the battery (State_Of_Charge) for each period of analysis. The State_Of_Charge is in the period ‘t’ is equal to the State_Of_Charge in period ‘t-1’ plus the energy flow into the battery, minus the energy flow out of the battery. This is done for each scenario i. In time t=1 the State_Of_Charge_Battery is equal to a fully charged battery.

Parameters:model – Pyomo model as defined in the Model_creation library.
Constraints.Total_Finalcial_Cost(model)¶

This funtion calculate the total financial cost of the system.

Parameters:model – Pyomo model as defined in the Model_creation library.
